# 📺 Emby-TG-Notifier 项目完全手册

## 1. 项目简介

**Emby-TG-Notifier** 是一款为 Emby 深度定制的 Telegram 通知系统。它通过 Python Flask 接收 Emby 的 Webhook 信号，并利用 TMDB API 自动补全元数据（如横屏海报、主演、简介等），同时通过调用 Emby 本地 API 实现精准的“已播出缺集”统计和“下一集”播出预报。

本项目专为解决“多集入库刷屏”、“同名剧集识别错误”以及“缺集统计不准”等痛点而生。

---

## 2. 核心功能特性

* **🚀 智能防抖聚合**：内置 10 秒处理队列。当一次性入库多集时，系统会自动合并为一条消息（例如：`S01E01-E10`），有效防止 Bot 刷屏。


* **🔍 三级 ID 溯源**：
* **Level 1**：直接读取 Webhook 携带的 TMDB ID。


* **Level 2**：若 Webhook 缺失，脚本会延迟 3 秒主动向 Emby API 反查，等待刮削完成。


* **Level 3**：若反查失败，使用“名称 + 年份”锁定搜索，彻底解决同名剧集乱跳问题。




* **🧮 精准缺集判定**：不仅对比本地库，还会比对 TMDB 的官方排播表。仅统计“已播出”但“本地缺失”的集数，未来未播集不计入缺集。


* **🗓️ 智能下集预报**：实时显示下一集的编号及播出日期（MM-DD）。若本地已追平进度，则显示“本季完结”。


* **🖼️ 动态横屏海报**：自动抓取 TMDB 1280px 宽度的横幅背景图，作为 Telegram 图文卡片封面。


* **🖥️ 质量标签识别**：通过正则引擎从文件名中提取 `DV`、`HDR`、`2160p` 等视频特征标签。


* **🔗 隐藏式链接**：卡片底部集成 TMDB、豆瓣、IMDb 的跳转链接，豆瓣链接支持通过 IMDb ID 精准搜索。

## 3. 环境参数配置 (Environment)

| 变量名 | 描述 | 获取/说明 |
| --- | --- | --- |
| `TG_BOT_TOKEN` | Telegram 机器人令牌 | 通过 @BotFather 获取

|
| `TG_CHAT_ID` | 接收通知的频道/群组 ID | 机器人必须已加入该群组

|
| `TMDB_API_KEY` | TMDB API 密钥 | 在 TMDB 官网设置页面申请

|
| `EMBY_SERVER` | Emby 内网/公网地址 | 结尾不要带 `/`，例如 `[http://192.168.1.100:8096](http://192.168.1.100:8096)`<br> |
| `EMBY_API_KEY` | Emby API 密钥 | Emby 后台 -> 设置 -> API 密钥

|
| `TZ` | 系统时区 | 默认 `Asia/Shanghai`<br> |
| `PYTHONUNBUFFERED` | 日志实时刷新 | 固定值 `1`<br> |

## 4. 部署及 Emby 配置步骤

### 第一步：启动容器

在目录下运行命令：

```bash
docker-compose up -d
```

### 第二步：Emby Webhook 配置

1. 登录 Emby -> 设置 -> Webhook -> **添加 Webhook**。


2. **Url**: 填写 `http://你的IP:18089/webhook`。


3. **触发事件**: 勾选 `库已更新 (library.new)` 或 `项目已添加 (Item Added)`。


4. **关键设置**：**务必取消勾选**“按剧集和专辑对通知进行合并分组”。


* *说明：取消该选项后，脚本才能接收到具体的单集路径并提取视频质量标签；多集聚合由脚本内置的 10 秒队列自动完成*。

## 6. 常见问题排查 (FAQ)

* **Q: 为什么“缺集”包含了未来还没出的集数？**
* A: 本项目代码已通过 `last_episode_to_air` 进行锁定，如果仍有此问题，请检查 TMDB 网页端该剧集的 Season 列表是否有人误填了未来日期。




* **Q: 下集预告显示为“待定”？**
* A: 这通常意味着 TMDB 数据库尚未填入下一集的播出日期，尤其是国产动画常见此情况。




* **Q: 电影入库没有海报？**
* A: 确认 Emby 已经为该电影刮削到了 TMDB ID，脚本会自动尝试反查 3 秒以等待刮削结果。
